PYTHON

💡 What is Python?

Python is a dynamically typed, strongly typed, interpreted language with automatic memory management. Source code → bytecode → executed by Python Virtual Machine (PVM).

❌ Trap

"Python is compiled like C" → WRONG. Python source → bytecode → PVM. Errors are mostly runtime, not compile-time.

PropertyMeaningTrap
Dynamically typedType decided at runtimeVariable ≠ fixed type
Strongly typedNo implicit unsafe coercion"5"+5 → ❌ TypeError
Everything is objectEven int, boolid() works on all
Reference-basedVariables hold referencesAssignment ≠ copy
🗃️ Data Types▶
TypeMutableNotes

float❌IEEE 754
bool❌Subclass of int!
str❌Sequence of Unicode
NoneType❌Singleton object
⚠️ Classic Trap
True + True  # Output: 2  (bool → subclass of int)
🔗 Variables & Identity▶

A variable is a name bound to an object reference, not a memory container.

# Small int caching
a = 10; b = 10
a is b   # True (small int cache)

a = 500; b = 500
a is b   # False (implementation dependent)
❌ Trap
  • Never use is for value comparison — use ==
  • is checks identity (same object), == checks equality (same value)
🔢 Operators — Key Traps▶
5 / 2    # 2.5  (true division)
5 // 2   # 2   (floor division)
-5 // 2  # -3  ⚠️ FLOOR, not truncate!
⚠️ Trap

-5 // 2 = -3, not -2! Python floors toward negative infinity.

🔄 Flow Control & Loops▶
Truthy / Falsy Values (MCQ Gold)

Falsy: 0, 0.0, '', None, False, [], (), {}

if []:
    print("Yes")
else:
    print("No")   # Output: No
else with Loops (VERY HIGH TRAP)
for i in range(3):
    if i == 5: break
else:
    print("Done")  # Output: Done (no break happened)

for i in range(3):
    if i == 1: break
else:
    print("Done")  # Output: (nothing — break occurred)
✅ Rule

else after a loop executes only if the loop ends normally (no break).

Strings & Tuples

Immutable sequences — slicing, formatting, packing, unpacking.

String

An immutable, ordered sequence of Unicode characters. Indexed from 0, supports slicing and iteration.

Immutable Ordered Hashable

Tuple

An immutable, ordered collection of heterogeneous objects. Supports packing/unpacking. Faster & lighter than lists.

Immutable Ordered
✂️ String Slicing (HIGH TRAP)▶
s = "abcdef"
s[1:4]    # 'bcd'
s[::-1]   # 'fedcba' (reversed)
s[5:1]    # '' (empty — NO IndexError)
s[1:10]   # 'bcdef' — slicing NEVER errors
✅ Key Rule

Slicing never raises IndexError — returns empty string if out of range.

Code Output Question
s = "Python"
print(s[1:5][::-1])
# s[1:5] → "ytho"  →  reversed → "ohty"

📦 Tuple Creation Traps▶
t = (1)    # ← This is an INT!
t = (1,)   # ← This is a TUPLE ✔
t = 1, 2, 3  # tuple packing
🔴 Most Common Trap

A comma creates a tuple, not parentheses! (1) is an int. (1,) is a tuple.

Mutable Object Inside Tuple
t = (1, [2, 3])
t[1].append(4)
print(t)   # (1, [2, 3, 4]) ← ALLOWED!
✅ Explanation

Tuple immutability applies to the container, not to mutable objects inside it. The inner list can still be mutated.

Session 3

Python Dictionaries

Mutable key-value mapping — hash tables, collision, access patterns.

Definition

A dictionary is a mutable mapping object storing key–value pairs, where keys are unique and hashable, implemented using a hash table. From Python 3.7+, insertion order is preserved.

🔑 Key Rules & Access▶
# Duplicate keys → last wins
d = {1: "A", 1: "B"}   # → {1: 'B'}

# Unhashable key → TypeError
d = {[1,2]: "x"}      # ❌ TypeError: unhashable list

# Safe access
d.get("z", 0)         # Returns 0, not KeyError
d["z"]               # ❌ KeyError
⚠️ True == 1 Trap (Classic!)
d = {1: "A", True: "B"}
print(d)   # {1: 'B'} — True==1, same hash!
📝 Reference Copy Trap▶
d = {"x": 1}
x = d
x["y"] = 2
print(d)   # {'x': 1, 'y': 2} — same object!
❌ Trap

Dictionary assignment copies the reference, not the object. Both d and x point to the same dict.

📊 dict vs list

Featuredictlist
AccessBy KeyBy Index
Duplicate keys❌✔
LookupO(1)O(n)

Session 4

Working with Lists

Mutable sequences — reference semantics, copy traps, mutations.

Definition

A list is a mutable, ordered sequence storing references to objects, allowing duplicates and dynamic resizing.

⚡ + vs += (CLASSIC CDAC TRAP)▶
# + creates NEW list
a = [1,2,3]; b = a
a = a + [4]
print(b)   # [1, 2, 3] — b still points to old list

# += mutates IN-PLACE
a = [1,2,3]; b = a
a += [4]
print(b)   # [1, 2, 3, 4] — same object!
🔴 Key Distinction
  • a + [4] → creates new list, b is unaffected
  • a += [4] → mutates in-place, b also changes
🔁 Repetition with Mutable (VERY HIGH TRAP)▶
# DANGEROUS
a = [[0]] * 3
a[0][0] = 5
print(a)   # [[5], [5], [5]] — same inner list!

# SAFE
a = [[0] for _ in range(3)]
⚠️ Explanation

* 3 repeats the same reference. Modifying one modifies all three.

📊 List vs Tuple

FeatureListTuple
Mutable✔❌
Hashable❌Conditional
MemoryHigherLower
SpeedSlowerFaster

🧠 Revision Chips

= → no copy[:] → shallow copy+ → new list+= → in-place* → reference repeatappend → mutates
Sessions 5 & 6

Functions & Methods

Scope, mutability, default arguments, LEGB rule.

Definitions

Parameter → variable in function definition. Argument → value passed during call. Arguments are passed by object reference (call by object sharing).

🔥 Mutable Default Argument (HIGHEST TRAP)▶
def f(x, lst=[]):
    lst.append(x)
    return lst

print(f(1))   # [1]
print(f(2))   # [1, 2] ← NOT [2]!
🔴 Explanation

Default list is created ONCE at definition time and shared across all calls.

✅ Safe Pattern
def f(x, lst=None):
    if lst is None:
        lst = []
    lst.append(x)
    return lst
🔬 UnboundLocalError Trap▶
x = 10
def f():
    print(x)  # ❌ UnboundLocalError!
    x = 20
f()
⚠️ Why?

Any assignment inside a function makes Python treat that variable as local at compile time. So print(x) references an unbound local.

🧬 Mutability in Functions▶
TypeCan mutate inside fnEffect outside
list✔Visible
dict✔Visible
int❌Not visible
str❌Not visible
# Mutable list — mutation visible outside
def f(lst): lst.append(4)
a = [1,2,3]; f(a)
print(a)   # [1, 2, 3, 4]

# Immutable int — reassignment not visible
def f(x): x = x + 1
a = 10; f(a)
print(a)   # 10

Tuples — Complete Guide

Every tuple trap, edge case, and hashability rule.

📦 Creation Forms & Single-Element Trap▶
t1 = (1, 2, 3)
t2 = 1, 2, 3        # tuple packing
t3 = tuple([1,2,3])
t4 = tuple("abc")   # ('a','b','c')

# SINGLE ELEMENT TRAP
t = (1)    # type: int
t = (1,)   # type: tuple ✔
🔁 Repetition with Mutable (VERY HIGH)▶
t = ([0],) * 3
t[0].append(1)
print(t)   # ([0, 1], [0, 1], [0, 1])
⚠️ Same Reference Repeated

Repetition duplicates references, not objects.

📤 Unpacking & Extended Unpacking▶
a, b = (1, 2)       # OK
a, b = (1, 2, 3)   # ❌ ValueError: too many values
a, *b = (1, 2, 3, 4)  # a=1, b=[2,3,4]
(a, (b, c)) = (1, (2, 3))  # nested unpacking

# Swap uses tuple internally
a, b = b, a
🔑 Tuple as Dictionary Key▶
d = {(1, 2): "a"}      # ✔ Valid
d = {([1,2]): "a"}    # ❌ TypeError
✅ Rule

A tuple is hashable if and only if all its elements are hashable.

Sessions 8 & 9

OOP, Generators & Decorators

Classes, MRO, yield, wrappers — all trap zones mapped.

🏗️ Classes & Objects▶
class A:
    x = 10   # class attribute — SHARED
    def __init__(self):
        self.y = 20  # instance attribute — OWN

a = A(); b = A()
a.x = 99   # creates instance attribute, class unchanged
print(b.x)   # 10 (class attr still)
⚠️ Mutable Class Attribute Trap
class A:
    lst = []
a = A(); b = A()
a.lst.append(1)
print(b.lst)   # [1] — shared!
🌀 Generators (VERY HIGH WEIGHT)▶

A generator is a function that yields values lazily, suspending execution state between calls.

def f():
    yield 1
    yield 2

g = f()
next(g)   # 1
next(g)   # 2
next(g)   # ❌ StopIteration
FeatureGeneratorList
MemoryLowHigh
ExecutionLazyEager
Reusable❌✔
🎁 Decorators (TRAP ZONE)▶

A decorator wraps another function, modifying behavior without changing source code.

def dec(fn):
    def wrapper():
        print("before")
        fn()
        print("after")
    return wrapper

@dec
def f(): print("hi")
f()
Output
before
hi
after
Key Point

@dec is equivalent to f = dec(f). Decorator runs at definition time, not call time.

🌳 MRO & super()▶

super() refers to the next class in MRO, not just the parent. For multiple inheritance, Python uses C3 linearization.

class D(B, C): pass
D.mro()   # [D, B, C, A, object]

🧠 Revision Chips

self → not a keywordclass attr → sharedsuper → MROyield → pausegenerator → lazydecorator → runs at def

🔄 finally Traps (VERY HIGH)▶
# finally overrides return
def f():
    try:
        return 1
    finally:
        return 2
print(f())  # 2 ← finally wins!

# finally suppresses exception
def f():
    try:
        1 / 0
    finally:
        return 10
print(f())  # 10 ← exception LOST!
⚠️ finally Rules
  • Always executes, even with return/break/exception
  • If finally has a return, it overrides try's return
  • If finally has a return, it suppresses exceptions
📝 else with try▶
try:

except:
    print("error")
else:
    print("success")  # ← runs if NO exception

NumPy, Pandas & Data Cleaning

Vectorization, views vs copies, alignment, missing values.

🔢 NumPy — Views vs Copies (CRITICAL)▶
import numpy as np
a = np.array([1,2,3,4])
b = a[1:3]   # ← VIEW, not copy!
b[0] = 99
print(a)   # [ 1 99  3  4] — original changed!

# Safe copy
b = a[1:3].copy()
⚠️ NumPy vs Python List

Python list slicing → shallow copy. NumPy slicing → VIEW (shares memory).

📡 Broadcasting Rules▶
Rules
  • Align shapes from right
  • Dimensions must be equal OR one of them is 1
  • Otherwise → ValueError
a = np.array([1,2,3])
b = np.array([[10],[20],[30]])
a + b   # (3,) + (3,1) → (3,3) ✔

np.array([1,2]) + np.array([1,2,3])  # ❌ ValueError
🐼 Pandas — Index Alignment (Silent Bug)▶
s1 = pd.Series([1,2,3], index=["a","b","c"])
s2 = pd.Series([10,20,30], index=["b","c","d"])
s1 + s2
Output

⚠️ Alignment by Label

Pandas aligns by index label, not position. Mismatched labels produce NaN.

🔍 loc vs iloc (EXTREMELY IMPORTANT)▶
df.loc[0]      # label-based
df.iloc[0]     # position-based
df["A"]        # Series (single column)
df[["A"]]      # DataFrame (list of columns)

# Label trap
s = pd.Series([1,2,3], index=[2,1,0])
s[0]   # 3 — label 0, not position 0!
🧹 Chained Indexing (SettingWithCopyWarning)▶
# WRONG — may not modify original
df[df["A"] > 1]["B"] = 0

# CORRECT
df.loc[df["A"] > 1, "B"] = 0

⚠️ Autocommit is OFF by default

After INSERT/UPDATE/DELETE, you must call commit() or data won't save. rollback() undoes uncommitted changes.

Session 19 (R)

R Foundations

Vectorization, recycling rule, 1-based indexing, packages.

R LANGUAGE

What is R?

R is an open-source, interpreted programming language for statistical computing and graphics. RStudio is an IDE, not the language itself.

⚠️ Trap

R is interpreted, not compiled. RStudio ≠ R.

🔄 Recycling Rule (HIGH TRAP)▶
c(1,2,3,4) + c(1,2)
# → 2 4 4 6   (shorter vector recycled)
# ⚠️ Warning: longer not multiple of shorter
⚠️ Recycling

R repeats shorter vector elements cyclically. Produces a warning if lengths aren't multiples.

📦 Package Installation vs Loading▶
install.packages("ggplot2")  # downloads to disk
library("ggplot2")           # loads into session
⚠️ Trap

install.packages() alone is not enough — you must also call library() before using the functions.

R Functions & R Markdown

Lazy evaluation, scope, na.rm, reproducible reporting.

⏳ Lazy Evaluation▶
f <- function(x) 10
f(stop("error"))   # Works! Arg never evaluated
Lazy Evaluation

R arguments are only evaluated when actually used. If not used, errors in arguments are never triggered.

📐 NA Propagation Trap▶
mean(c(1, NA, 3))                # NA
mean(c(1, NA, 3), na.rm=TRUE)   # 2
📄 R Markdown▶
⚠️ Knitting Trap

During knitting, the global environment is reset. Code that worked interactively may fail if it depends on variables set earlier in a session but not in the document.

R Markdown

Chunks execute top-to-bottom. Cached chunks may skip execution. Always ensure all dependencies are defined within the document.
